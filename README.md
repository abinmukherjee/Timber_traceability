## Setup & Deployment Guide

### Prerequisites

- Node.js 18+, npm
- Java 17
- Maven
- PostgreSQL (running locally on port 5432)
- MetaMask installed in browser with Sepolia ETH
- Alchemy account (for Sepolia HTTP + WebSocket RPC URLs)
- Pinata account (for IPFS uploads)

---

### Step 1: Smart Contract

```bash
cd contracts

# Copy and fill in your environment variables
cp .env.example .env
# Edit .env: add SEPOLIA_RPC_URL, PRIVATE_KEY, ETHERSCAN_API_KEY

# Install dependencies
npm install

# Run tests (must all pass before deploying)
npx hardhat test

# Deploy to Sepolia
npx hardhat run scripts/deploy.js --network sepolia
# в†’ Outputs: вњ… TimberSupplyChain deployed to: 0x...
# в†’ Writes contract-config.json (project root + frontend/js/)
```

> **Note:** Copy the deployed address вЂ” you'll need it for Step 2 and 3.

---

### Step 2: PostgreSQL Database

```sql
-- Create the database (once)
CREATE DATABASE timber_db;
```

Spring Boot JPA will auto-create all tables on first run (`ddl-auto=update`).

---

### Step 3: Spring Boot Backend

```bash
cd backend

# Edit application.properties:
#   spring.datasource.password=YOUR_POSTGRES_PASSWORD
#   contract.address=0x...  (from Step 1)
#   pinata.jwt=...
#   app.base.url=http://localhost:5500  (or your frontend URL)

mvn spring-boot:run
# в†’ Starts on http://localhost:8080
# в†’ Tables auto-created in PostgreSQL
```

**Verify:** `curl http://localhost:8080/api/stats` в†’ `{"lots":0,"batches":0,"furnitures":0,"roles":0}`

---

### Step 4: ethers.js Event Listener

```bash
cd listener

# Copy and fill in environment variables
cp .env.example .env
# Edit .env: SEPOLIA_WS_URL (wss://...), CONTRACT_ADDRESS (from Step 1)

npm install
npm start
# в†’ Connects to Sepolia WebSocket
# в†’ Listening for: LotRegistered, WoodPurchased, FurnitureCreated, RoleGranted, RoleRevoked
```

---

### Step 5: Frontend (React SPA)

```bash
cd frontend
npm install
npm run dev        # http://localhost:5500 (falls back to a free port if taken)
```

For a production build: `npm run build` (outputs `dist/`), or run the whole stack with
`docker compose up` (the `frontend` service builds and serves via nginx on port 5500).

> After deploying the contract, `deploy.js` automatically writes `frontend/public/contract-config.json`
> with the real address and ABI. The app fetches it at startup вЂ” swap the file to re-point at a new
> contract with no rebuild. Override the backend URL with `VITE_API_BASE` (e.g. in `frontend/.env`).

### Step 6: End-to-End Demo Flow

1. **Admin** в†’ `admin.html` в†’ Connect MetaMask (deployer) в†’ Grant `AUCTION_HOUSE_ROLE` to Wallet A в†’ Grant `FACTORY_ROLE` to Wallet B в†’ `roles` table updates via listener
2. **Wallet A** в†’ `auction.html` в†’ Register Lot #1 (Teak, 2000 cft, Grade A) в†’ appears in dashboard
3. **Wallet B** в†’ `factory.html` в†’ Tab 1: Buy 100 cft from Lot #1 в†’ Batch #1 created
4. **Wallet B** в†’ `factory.html` в†’ Tab 2: Upload certificate to Pinata в†’ Create Furniture #1 (Bed, 40 cft)
5. **Anyone** в†’ `verify.html?id=1` в†’ Full provenance timeline: Furniture в†’ Batch в†’ Lot в†’ Forest Origin
6. **Wallet C (no role)** в†’ attempts `registerLot` в†’ MetaMask submits в†’ contract reverts в†’ nothing written to DB

## Critical Design Rules

1. **Backend never holds a private key** вЂ” all transactions signed by MetaMask
2. **IPFS upload always before chain transaction** вЂ” CID is a constructor argument
3. **Every state-changing function has `onlyRole()` modifier**
4. **Conservation `require()` checks before any state mutation**
5. **Event listener pipeline fires only on mined events** вЂ” no pending tx handling
6. **Database is a read-optimisation mirror** вЂ” on-chain is source of truth
7. **Provenance via object linkage** (`parentLotId`, `sourceBatchId`) вЂ” never balance-based

## API Reference

### Public (frontend)
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/stats` | Dashboard counts |
| GET | `/api/lots` | All lots |
| GET | `/api/lots/{id}` | Lot detail |
| GET | `/api/batches/{id}` | Batch detail |
| GET | `/api/batches?factory=0x...` | Batches by factory |
| GET | `/api/furnitures/{id}` | Furniture detail |
| GET | `/api/verify/{id}` | Full provenance chain (QR scan endpoint) |
| GET | `/api/roles` | Role assignments |
| GET | `/api/qr/{id}` | QR code PNG image |

| Method | Endpoint | Description |
|---|---|---|

| POST | `/api/internal/furnitures` | Mirror FurnitureCreated event |
| POST | `/api/internal/roles/grant` | Mirror RoleGranted event |
| POST | `/api/internal/roles/revoke` | Mirror RoleRevoked event |
